# PCB 完整讲解（lwIP 里的 udp_pcb）
## 1. 全称
**PCB = Protocol Control Block**
中文：**协议控制块**

## 2. 通俗一句话解释
操作系统 / lwIP 内核用来**记录一条UDP/TCP连接/套接字所有状态、配置、地址信息**的专属结构体，每打开一个UDP套接字，内核就分配一块PCB内存单独管理它。
你可以把 PCB 理解成：**UDP套接字在内核里的档案本**。

## 3. 为什么需要 PCB？
一个程序可以同时开多个UDP端口、多个远端通信对象，lwIP 必须区分开每一路通信：
- 哪个本地IP、本地端口
- 连的哪个远端IP、远端端口
- 收数据时交给哪个应用回调函数
- TTL、广播权限、UDPLite、组播配置
- 绑定的网卡、缓存标记等

所有这些信息全部打包存在 `struct udp_pcb` 里，就是UDP PCB。

## 4. 结合你源码里的 `struct udp_pcb` 包含什么档案
1. **链表指针 next**
全局有一条链表 `udp_pcbs`，所有活跃UDP PCB串在一起，收包时遍历链表匹配对应套接字。
2. **本地端信息**
`local_ip`、`local_port`：bind绑定的IP和端口。
3. **远端信息**
`remote_ip`、`remote_port`：connect指定的对端地址。
4. **接收回调**
`recv`、`recv_arg`：收到数据后调用应用注册的函数，把数据包交给程序。
5. **状态标志 flags**
是否connected、是否UDPLite、是否允许组播回环、是否关闭校验和。
6. **网络配置**
TTL、TOS、绑定网卡索引、组播专用参数、UDPLite校验长度等。

## 5. 生命周期（对应源码函数）
1. `udp_new()`：分配一块新PCB（新建空白档案）
2. `udp_bind()`：填写本地IP端口，把PCB挂入全局链表
3. `udp_connect()`：填写远端IP端口，标记为连接态
4. `udp_recv()`：登记应用接收回调函数
5. `udp_send()`/`udp_sendto()`：读取PCB里的地址配置发包
6. `udp_input()`：收到报文，遍历PCB链表匹配，找到对应PCB投送数据
7. `udp_remove()`：销毁PCB，释放内存，从链表摘除

## 6. 关键区分
1. UDP PCB 是**无连接也存在**的
UDP本身无连接，哪怕不调用`connect`，只要bind端口，就会生成PCB监听端口。
`connect`只是在PCB里记录远端地址，不会发网络包，只是内核层面标记。

2. 和TCP PCB区别
TCP PCB会额外保存序列号、滑动窗口、重传队列、连接状态机(ESTABLISHED/SYN_SENT等)；
UDP PCB极简，只存地址、回调、配置，没有复杂传输状态。

## 7. 类比举例
- 你开一个UDP服务监听8080端口 → 创建1个udp_pcb档案
- 再开客户端连192.168.1.100:9000 → 再创建1个独立udp_pcb
- 网卡收到8080端口数据 → lwIP遍历PCB链表，匹配到监听8080的那块PCB，调用你注册的recv函数递数据。

## 8. 核心总结
PCB是协议栈内核管理单条通信通道的**状态结构体**；
udp_pcb就是专门给UDP套接字使用的协议控制块，所有收发、绑定、连接逻辑都围绕它操作。
