基于你前面已经熟悉了 lwIP 的目录结构和理论书籍，现在正式进入 **动手实践阶段**。

本指南以 **Linux (WSL2 或原生 Ubuntu)** 为例，覆盖 **环境准备 → 编译 → 运行 → 调试** 的完整流程。Windows (MinGW) 环境除了 TAP 网卡配置差异外，后续 CMake 步骤完全通用。

## 📦 1. 环境准备

### 1.1 安装基础工具链

```bash
sudo apt update
sudo apt install -y git make cmake gcc g++    # 核心编译工具
sudo apt install -y bridge-utils uml-utilities   # tap 网卡管理工具
```

`bridge-utils` 用于管理网络桥接，`uml-utilities` 提供 `tunctl` 命令创建虚拟网卡。

### 1.2 安装调试工具

```bash
sudo apt install -y gdb          # 源码级调试
sudo apt install -y tcpdump      # 命令行抓包
```

Wireshark（图形化抓包分析）建议在 **宿主机** 上安装，用于可视化分析 TCP 状态变迁。安装完成后，确保你的用户有 `sudo` 权限。

### 1.3 验证编译工具版本

```bash
cmake --version          # 建议 ≥ 3.10
gcc --version            # 建议 9.x ~ 11.x
```

CMake 是构建 lwIP 的核心工具，如果版本过低可以升级：

```bash
sudo apt install -y cmake=3.22.1-*   # 或从源码编译新版
```


## 🔧 2. 获取源码

**⚠️ 重要**：不要单独下载旧的 `contrib-2.1.0.zip`，其 API 和目录结构与新版不兼容。直接从 **GitHub** 克隆最新的 `lwip` 和 `lwip-contrib` 仓库。

```bash
cd ~
git clone https://github.com/lwip-tcpip/lwip.git
git clone https://github.com/lwip-tcpip/lwip-contrib.git
```

验证目录结构：

```bash
ls ~/lwip/src/core/        # 看到 tcp.c, tcp_in.c, tcp_out.c 等
ls ~/lwip-contrib/ports/unix/   # 看到 example_app/ minimal/ 等
```


## 🏗️ 3. 编译 example_app（官方推荐的工作示例）

这是 lwIP **官方推荐** 的入门示例，运行一个功能完整的 echo server，整合了 TCP/UDP echo、iperf 等多种测试功能。

### 3.1 复制配置文件

```bash
cd ~/lwip-contrib/examples/example_app
cp lwipcfg.h.example lwipcfg.h
```

`lwipcfg.h` 是网络接口配置文件，控制 IP 地址、DHCP、网卡名称等参数。

### 3.2 修改配置（根据自己的需求选择）

打开 `lwipcfg.h` 并做出以下修改：

```c
/* 第 50 行左右 —— 静态 IP 配置（推荐新手使用静态 IP） */
#define USE_DHCP       0    // 注释：#define USE_DHCP 1
#define USE_AUTOIP     0    // 注释：#define USE_AUTOIP 1

/* 第 54-56 行 —— 静态 IP 参数 */
#define MY_IPADDR      "192.168.1.200"   // lwIP 模拟设备的 IP
#define MY_NETMASK     "255.255.255.0"   // 子网掩码
#define MY_GATEWAY     "192.168.1.1"     // 网关（宿主机 tap 网卡的 IP）

/* 第 74 行 —— 启用 iperf 服务器功能（可选） */
#define LWIP_LWIPERF_APP  1
```

**📖 书本对应：** 《自顶向下》第 4.3.3 节（IP 地址、子网掩码和网关的概念）。静态 IP 配置对应书本中的“主机配置”部分——每台主机需要一个唯一的 IP 地址。

### 3.3 CMake 跨平台构建

```bash
cd ~/lwip-contrib/ports/unix/example_app
mkdir -p build && cd build

# 关键：必须通过 -DLWIP_DIR 告诉 CMake 核心源码的位置
cmake -DLWIP_DIR=~/lwip ..
make -j$(nproc)
```

**📖 书本对应：** 《自顶向下》第 2.7 节（套接字编程的编译与运行环境）。CMake 构建系统的原理与书本中“编写 Makefile 编译网络应用程序”的思路一致。

✅ 成功标志：编译完成后在 `build/` 目录下生成可执行文件 **`example_app`**。


## 🌐 4. TAP 虚拟网卡配置（关键步骤）

**📖 书本对应：** 《TCP/IP 详解》第 2.6 节（环回接口）。TAP 虚拟网卡在原理上类似于环回接口，但它提供了一个完整的数据链路层接口，让协议栈与宿主机网络栈隔离运行——这正是你“解剖”协议栈的最佳实验室环境。

### 4.1 创建 TAP 设备

⚠️ **注意**：`someuser` 必须替换为你的实际 Linux 用户名。你可以用 `whoami` 命令查看自己的用户名。

```bash
# 创建 tap0 设备，归属到你的用户名
sudo tunctl -u someuser -t tap0

# 启用网卡并配置 IP
sudo ifconfig tap0 192.168.1.1 up

# 开启 IP 转发（让 tap0 和宿主机其他网卡互通）
sudo sysctl -w net.ipv4.ip_forward=1
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE   # 若需要访问外网
```

### 4.2 验证网卡状态

```bash
ifconfig tap0
# 应该显示：
# tap0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
#         inet 192.168.1.1  netmask 255.255.255.0  broadcast 192.168.1.255
```

### 4.3 确认 lwipcfg.h 中的网卡名称

打开 `~/lwip-contrib/examples/example_app/lwipcfg.h`，找到并确认：

```c
#define LWIP_TAPIF                 "tap0"   // 与上一步创建的网卡名称一致
```

### 4.4 以 root 权限运行 example_app

```bash
cd ~/lwip-contrib/ports/unix/example_app/build
sudo ./example_app
```

✅ 成功标志：看到类似以下的输出，表示 lwIP 已经启动并绑定到 TAP 网卡：

```
Bringing up network interface...
ifconfig tap0 192.168.1.200 netmask 255.255.255.0 up
ifconfig tap0
tap0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 192.168.1.200  netmask 255.255.255.0  broadcast 0.0.0.0
******  START **********
```


## 🔌 5. 测试网络连通性

### 5.1 Ping 测试

**终端 A (运行 lwIP)**：`sudo ./example_app`（保持运行）

**终端 B (宿主机)**：

```bash
ping 192.168.1.200
```

✅ 预期：`64 bytes from 192.168.1.200: icmp_seq=1 ttl=64 time=0.xxx ms`

**📖 书本对应：** 《TCP/IP 详解》第 7 章（ICMP 协议）。每次 ping 都产生一个 ICMP Echo Request 报文，lwIP 内核中的 `icmp_input()` 函数解析它并回复 Echo Reply。你可以对照书本第 7.2 节的 ICMP 报文格式图，在源码中找到每个字段的解析逻辑。

### 5.2 Echo Server 测试

```bash
# TCP Echo：端口 7
nc 192.168.1.200 7
输入: hello
预期: hello (原样返回)

# UDP Echo：端口 7
nc -u 192.168.1.200 7
```

**📖 书本对应：** 《自顶向下》第 3.3 节和第 3.5 节（UDP/TCP 的 Echo 服务）。echo server 的代码在 `~/lwip-contrib/apps/tcpecho_raw/tcpecho_raw.c`，是学习 **Raw/Callback API** 的最佳范例。

### 5.3 Iperf 性能测试（可选）

**终端 A**：确保 `lwipcfg.h` 中 `LWIP_LWIPERF_APP` 宏定义为 1。

**终端 B**：

```bash
sudo apt install iperf
iperf -c 192.168.1.200 -p 5001    # lwIP 默认监听 5001 端口
```

这项测试会快速产生大量 TCP 数据流，非常适合 **观察滑动窗口和拥塞控制** 在实际传输中的行为。

**📖 书本对应：** 《自顶向下》图 3.39（滑动窗口演示）和 **《TCP/IP 详解》** 第 16 章（TCP 拥塞控制）。观察 iperf 输出中窗口大小的变化，对照源码 `tcp_out.c` 中 `tcp_output()` 的实现。

### 5.4 一键 setup 脚本（推荐）

lwIP 提供了自动化脚本，可以一键创建 tap 网卡、构建和运行：

```bash
cd ~/lwip-contrib/ports/unix
sudo ./setup-tapif     # 自动创建 tap0 网卡
```

这个方法比手动逐条输入命令更方便，尤其适合第一次运行时的快速验证。


## 🐛 6. 调试与代码阅读技巧

### 6.1 GDB 源码级调试（核心技能）

```bash
# 重新编译，加入调试符号
cd ~/lwip-contrib/ports/unix/example_app/build
cmake -DLWIP_DIR=~/lwip -DCMAKE_BUILD_TYPE=Debug ..
make

# GDB 调试
sudo gdb ./example_app
(gdb) b tcp_input           # 在 TCP 输入函数设置断点
(gdb) b tcp_process         # 在 TCP 状态机处理函数设置断点
(gdb) run
(gdb) bt                    # 查看调用栈
(gdb) p *pcb                # 打印 TCP 控制块状态
```

**断点策略建议：**

| 调试目标 | 推荐断点位置 | 可观察的数据 |
|---------|------------|------------|
| TCP 三次握手 | `tcp_process()` 中的 `SYN_SENT` 分支 | `pcb->state` 的变迁 |
| 数据收发 | `tcp_receive()` (tcp_in.c) | `pcb->rcv_nxt`, `pcb->snd_una` |
| 重传机制 | `tcp_rexmit_rto()` (tcp_out.c) | `pcb->rto`, `pcb->nrtx` |
| 窗口更新 | `tcp_update_rcv_ann_wnd()` | `pcb->rcv_ann_wnd` |

### 6.2 启用调试宏打印

在 `lwipcfg.h` 开头添加：

```c
#define LWIP_DEBUG    1
#define TCP_DEBUG     LWIP_DBG_ON     // 打印 TCP 状态变化
#define PBUF_DEBUG    LWIP_DBG_ON     // 打印数据包结构
#define MEM_DEBUG     LWIP_DBG_ON     // 打印内存分配
```

`LWIP_DBG_ON` 开启后，运行时会输出详细的协议栈内部信息。你可以对照《TCP/IP 详解》中的抓包分析图，**同时观察控制台打印的状态变迁和 Wireshark 的报文**，这种“源码 + 日志 + 抓包”三位一体的方式能极大加深理解。

### 6.3 用 Wireshark 抓包分析（必做）

在 **宿主机** 上打开 Wireshark，抓取 `tap0` 网卡：

```bash
sudo tcpdump -i tap0 -w lwip_trace.pcap    # 命令行抓包保存
```

随后在 Wireshark 中打开 `lwip_trace.pcap`，筛选 `tcp`，可以清晰地看到：

- **三次握手**的报文序列
- **数据段**的序列号变化（对比 `tcp_out.c` 中 `tcp_write()` 的序列号分配）
- **滑动窗口**的更新
- **拥塞窗口**的增长过程（对照 `tcp_slowtmr()` 的实现）


## 🧪 7. 动手实践：追踪 TCP 三次握手

现在，让我们动手实验来追踪 TCP 三次握手。这是理解 lwIP 源码最直观的练习。

| 步骤 | 操作 | 观察目标 | 书本对照 | 源码定位 |
|------|------|---------|---------|---------|
| 1 | `sudo ./example_app` | 服务端启动，`tcp_bind()` 绑定端口 7 | 《TCP/IP 详解》13.2 节 | `tcpecho_raw.c` → `tcp_new()` → `tcp_bind()` |
| 2 | Wireshark 开始捕获 `tap0` | — | — | — |
| 3 | `nc 192.168.1.200 7` | 观察 Wireshark 的 SYN → SYN-ACK → ACK 三次握手 | 《自顶向下》图 3.39（状态变迁） | `tcp_in.c` 中 `tcp_process()` 的状态分支 |
| 4 | 输入 `hello` 并按回车 | 观察数据段的序列号和确认号变化 | 《TCP/IP 详解》13.3 节 | `tcp_out.c` 的 `tcp_write()` 和 `tcp_enqueue()` |
| 5 | `Ctrl+C` 中断 nc | 观察 FIN → FIN-ACK → ACK 四次挥手 | 《TCP/IP 详解》13.2 节 | `tcp_in.c` 中的 `TCP_FIN_WAIT1` 等状态处理 |

**进阶任务**：完成这个实验后，可以将 `lwipcfg.h` 中的 `USE_DHCP` 改为 1，移除 tap 网卡的静态配置，**观察 DHCP 协议如何分配 IP**。这对应《TCP/IP 详解》第 16.3 节（DHCP 协议）。


## 📚 8. 常见问题排查（FAQ）

| 问题现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `cmake ..` 失败，提示 `Could NOT find Doxygen` | 缺少 Doxygen 文档生成工具 | `sudo apt install doxygen` |
| `make` 时找不到 `lwip/opt.h` | CMake 的 `-DLWIP_DIR` 路径错误 | 确认 `ls ~/lwip/src/include/lwip/opt.h` 存在，重新 `cmake -DLWIP_DIR=正确路径` |
| `sudo ./example_app` 报错 "Cannot open tap device" | TAP 网卡未创建或用户权限不足 | `sudo tunctl -u $(whoami) -t tap0 && sudo ifconfig tap0 up` |
| Ping 不通 192.168.1.200 | 网关未配置或 IP 转发未开启 | 检查 `lwipcfg.h` 中 `LWIP_TAPIF` 名称、网关地址、`sudo sysctl net.ipv4.ip_forward=1` |
| 开启 DEBUG 打印后输出过多 | 所有调试宏都打开了 | 只保留需要的宏（如 `TCP_DEBUG`），其他设为 `LWIP_DBG_OFF` |
